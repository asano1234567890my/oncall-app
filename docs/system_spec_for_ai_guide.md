## 1. アプリ概要

病院の当直・日直スケジュールを自動生成するWebアプリ。

- **フロントエンド**: Next.js 16 + React 19 + TypeScript + Tailwind CSS
- **バックエンド**: Python FastAPI（非同期）
- **最適化エンジン**: Google OR-Tools CP-SAT ソルバー
- **DB**: PostgreSQL（Neon クラウド）
- **デプロイ**: Vercel（フロント）+ Render（バックエンド）

### 画面構成

| パス | 用途 | 対象ユーザー |
|------|------|------------|
| `/app` | モバイル版メイン画面（セットアップウィザード→設定→生成→D&D編集→保存） | 管理者 |
| `/dashboard` | PC版メイン画面（左パネル設定+右パレット+中央シフト表） | 管理者 |
| `/entry/[token]` | 個別医師の不可日入力（マジックリンク） | 各医師 |
| `/entry/shared/[token]` | 共有入力（1URLで全医師がドロップダウンから入力） | 各医師 |
| `/view` | 確定シフト閲覧（管理者ログイン） | 管理者 |
| `/view/[token]` | 確定シフト閲覧（医師個人） | 各医師 |
| `/report` | レポート・分析（スコア推移・回数マトリクス） | 管理者 |
| `/admin/doctors` | 医師管理（追加・編集・アーカイブ・ロック） | 管理者 |
| `/admin/dev` | 開発者管理（アカウント一覧・利用統計） | 開発者（is_superadmin） |

## 2. シフト生成の基本フロー

```
1. 管理者が医師を登録（名前・人数）
2. 各医師がマジックリンクで不可日を入力
3. 管理者が設定を確認（間隔・上限・モード等）
4. 「生成」ボタン → バックエンドのCP-SATソルバーが最適化
5. 結果をD&D（ドラッグ&ドロップ）で手動調整
6. 「確定保存」→ DBに保存 → 医師に公開
```

## 3. シフトの種類とスコア

### シフト種別

| 種別 | いつ | スコア（デフォルト） |
|------|------|-------------------|
| 平日当直 | 月〜金の夜間 | 1.0 |
| 土曜当直 | 土曜の夜間 | 1.5 |
| 日祝日直 | 日曜・祝日の午前 | 0.5 |
| 日祝当直 | 日曜・祝日の夜間 | 1.0 |
| 日当直（combined） | 日曜・祝日の日直+当直を同一人物 | 1.5 |

### スコアの意味
- スコア = その医師の月間の負担量
- 全医師のスコアが均等になるようにソルバーが最適化する
- スコアの下限（score_min）と上限（score_max）を設定可能
- 医師ごとに個別のmin/max/targetも設定可能

## 4. 日当直モード（holiday_shift_mode）

### split（別々）モード — デフォルト
- 日曜・祝日に「日直」と「当直」を**別々の医師**が担当
- 1日に2人必要
- `prevent_sunhol_consecutive = true`（同一医師が日直+当直を兼務できない）

### combined（日当直）モード
- 日曜・祝日に「日直」と「当直」を**同一医師**が担当
- 1日に1人で済む
- スコアは1.5pt（日直0.5 + 当直1.0の合算）
- `prevent_sunhol_consecutive` は自動OFF
- フロントエンドのD&D: day_shift + night_shift がペアで移動
  - 通常モード: ペア移動（day/nightどちらをドラッグしても両方移動）
  - 強制モード: 個別移動可能

### フロントエンドの制約チェック
- combinedモードの祝日ではday_shiftの検証をスキップ（night_shiftのコピーなので）
- `preventSunholConsecutive` はcombinedモード時に自動false
- 回数カウント（maxSunholWorks, maxWeekendHolidayWorks）はcombinedモード時にnightのみカウント（2倍カウント防止）

## 5. ハード制約（HardConstraints）

生成時に**必ず守られる**制約。違反するとスケジュールが生成できない（解なし）。

| 制約 | 設定キー | デフォルト | 説明 |
|------|---------|-----------|------|
| 勤務間隔 | `interval_days` | 4日 | 当直後に最低N日空ける |
| 土日祝合算上限 | `max_weekend_holiday_works` | 3回 | 1ヶ月あたりの土日祝勤務の合計上限 |
| 土曜当直上限 | `max_saturday_nights` | 1回 | 1ヶ月の土曜夜間当直の上限 |
| 日祝日直上限 | `max_sunhol_days` | なし | splitモード時のみ有効 |
| 日祝勤務上限 | `max_sunhol_works` | なし | 日祝の日直+当直の合計上限 |
| 不可日厳守 | `respect_unavailable_days` | ON | ONでハード制約、OFFでソフト制約 |
| スコア下限 | `score_min` | 0.5 | 全医師のスコアがこれ以上 |
| スコア上限 | `score_max` | 4.5 | 全医師のスコアがこれ以下 |

### 不可日のソフト化
- 個別不可日は `is_soft_penalty` フラグでソフト化可能
- ソフト化すると「なるべく尊重するが、解なし時は無視される」
- 固定不可曜日はソフト化の対象外（常にハード制約）
- 「全不可日をソフト化」ボタンで一括変更可能（医師管理画面）
- ソフト化中はトップページに琥珀色の警告バナーが表示される

---

## 6. ソフト制約と重み（ObjectiveWeights）

生成時に「できるだけ守りたい」制約。重みが大きいほど優先される。

| グループ | 子要素 | 説明 |
|---------|--------|------|
| 目標スコアへの近さ | target, score_balance | 各医師の目標スコアに寄せる |
| 土日祝の均等化 | sunhol_fairness, sat_month_fairness, past_sunhol_gap, past_sat_gap | 土日祝回数を医師間で均等に |
| 勤務間隔のゆとり | ideal_gap_weight + ideal_gap_extra | ハード間隔を超えた理想間隔への線形グラデーションペナルティ |

## 7. 外部枠（ダミー医師方式）

非常勤医師が入る日や、自チームの担当外の日を管理する仕組み。

### 仕組み
- DB上に「外部1」〜「外部31」のダミー医師を作成（`is_external=true`）
- 各ダミー医師は**当月ちょうど1回勤務**（枠数分が確実に外部に割り当てられる）
- スコア制約の対象外（min/max/targetが適用されない）
- 月マタギの間隔制約もスキップ（prev_month_worked_daysから除外）
- D&Dで常勤医師と入れ替え可能

### 2つの指定モード（external_input_mode）

ユーザーは「外部枠数で指定」か「勤務日数で指定」を選べる。

| モード | 設定値 | カレンダーデータ | 生成時変換 |
|--------|--------|---------------|-----------|
| 外部枠数で指定 | `external_slot_count` | `external_fixed_dates` | そのまま送信 |
| 勤務日数で指定 | `internal_day_count` | `internal_fixed_dates` | 補集合を`external_fixed_dates`に変換 |

- 両モードのデータはDB上に**独立保存**（切替時に互いのデータを変更しない）
- 「全日程」ボタンで外部枠なしに戻せる（external_slot_count=0）

### カレンダー操作
- 外部枠モード: タップした日がティール色（外部枠）
- 勤務日数モード: タップした日がブルー色（勤務日）
- 「全日勤務」/「全日外部」ボタンで一括操作
- 日曜・祝日はポップオーバーで日直/当直/終日を選択可能

### UI表示
- シフト表内: 「外部」とティール色で表示
- パレット: 外部枠エントリ（配置回数の自動集計）
- モバイルスコアサマリー: 外部枠の回数表示（タップでハイライト）
- レポート: 外部医師は集計から除外
- エクスポート: 「外部」名で出力

### 事前チェック
- 勤務枠数 < 常勤人数 かつ score_min > 0 → エラー
- スコア合計チェック: 外部枠分を差し引いた配分可能スコアで判定

### 保護
- 外部医師はDELETE不可（ソフト/ハード両方）
- bulk-lockの対象外
- 共有入力ドロップダウンに表示されない
- 医師管理画面に表示されない

## 8. 解なし時の診断システム

生成が「解なし」になった場合の診断フロー:

### Phase 1: 事前チェック（pre_validate）
生成ボタンを押した直後、ソルバーを動かす前に実行。

1. **容量チェック**: 医師数×(日数÷(間隔+1)) < 必要枠数 → 間隔を下げるか医師を増やす
2. **日別人手チェック**: 各日の不可日を考慮して勤務可能医師数が足りるか
3. **スコア下限チェック**: 勤務枠数 < 常勤人数 → スコア下限を満たせない
4. **スコア合計チェック**: 常勤の配分可能スコア vs 全員のスコア下限合計
5. **ロック済みシフト矛盾**: 不可日にロック済みシフトが存在

### Phase 2: AI診断（diagnose）
ソルバーが解なしを返した後に実行。

1. **管理者設定探索**: 各設定を1つずつ外してソルブ → 解ければ最小変更値を探索
   - 勤務間隔、土日祝合算上限、土曜上限、スコア上限、スコア下限
   - 組み合わせ探索（2設定同時）
2. **不可日の最小解除セット**: どの医師のどの不可日を外せば解けるか
3. **統計的観察（human_insights）**: 不可日が集中する日、不可日が多い医師

### 診断結果の表示
- 「検証済み解決策」として具体的な設定変更を提案
- 案内モーダルで解決方法をガイド

## 9. セットアップウィザード

初回利用時の9ステップウィザード:

1. **医師人数**: 2〜30名（±ボタン）
2. **医師名**: 手入力 or ファイル取込（Excel/Word/PDF/画像→Gemini OCR）
3. **当直形式**: 日当直（combined）/ 日直+当直別（split）
4. **当直枠の範囲**: 全日程 / 一部の日程のみ
   - 一部選択時: 「外部枠数で指定」（初期値4）/ 「勤務日数で指定」（初期値8）
5. **勤務頻度**: 最低回数（0〜4）/ 最大回数（1〜10）
6. **勤務間隔**: 0〜7日
7. **土曜上限**: 制限なし / 1回 / 2回
8. **不可日設定**: 今は設定しない / 先に設定する
9. **確認画面**: 全設定のサマリー → 「この設定で始める」

## 10. D&D（ドラッグ&ドロップ）操作

### PC版（DashboardScheduleTable）
- 右パレットから医師名をドラッグしてセルにドロップ → 配置
- セル間ドラッグ → 入れ替え
- ゴミ箱にドロップ → 解除
- 入れ替えボタン（ArrowLeftRight）でスワップ操作
- ロックボタンでシフト固定（固定したシフトは再生成時も維持）

### モバイル版（MobileScheduleBoard）
- 1タップ目: 医師をハイライト
- 2タップ目（同じセル）: アクションシート表示
  - 変更: 医師リストから選択（制約チェック付き）
  - 入替え: スワップモードに入る
  - 外す: シフト解除
  - 固定/解除: ロック切替
- 空セルタップ: 直接アクションシート

### 制約チェック
- D&D/入替え時にリアルタイムで制約チェック
- 違反時は赤色表示+エラーメッセージ
- 強制モード（Override Mode）: 全制約を無視して配置可能
- 外部医師は制約チェック対象外

### 保存時バリデーション
- 空欄チェック（当直未定の日）
- 間隔制約チェック
- 回数上限チェック
- 違反がある場合は警告表示 + 「違反を無視して確定する」オプション

## 11. 不可日（Unavailable Days）

### 個別不可日
- 医師ごとに特定の日に休み希望を設定
- target_shift: 「終日」「日直のみ」「当直のみ」
- is_soft_penalty: ソフト化フラグ（true=なるべく尊重、false=絶対厳守）
- マジックリンクで医師本人が入力可能

### 固定不可曜日
- 特定の曜日を毎週不可に設定（例: 毎週水曜は不可）
- 祝日も設定可能（day_of_week=7）
- target_shift で日直/当直/終日を指定
- ソフト化の対象外（常にハード制約）

### 不可日上限
- 管理者が「1ヶ月あたりN日まで」と制限可能
- KV設定 `unavail_day_limit` で管理

## 12. 前月の勤務実績

### 目的
- 月マタギの連続当直を防ぐ
- 過去のスコア・土曜回数・日祝回数を考慮した均等化

### データソース
- DB上の前月ShiftAssignment
- フロントエンドで `prevMonthWorkedDaysMap` として管理
- 外部医師は除外（月マタギ間隔チェック対象外）

## 16. 認証・マルチテナント

- JWT認証（施設名 or メールアドレス + パスワード）
- 新規登録時はメールアドレス必須
- 既存アカウントはメアド未登録でも利用可能（ログイン後にバナーで登録促進）
- メールアドレスはアカウント設定画面で確認・変更可能
- 病院名の変更はメアド登録済みの場合のみ可能
- hospital_idでマルチテナント分離
- 全データにhospital_idのFK
- マジックリンク: 医師ごとのaccess_tokenでURL認証（ログイン不要）
- is_locked: 入力締切後のロック（管理者が一括操作）
- ログイン後のリダイレクト先は画面幅で自動判定（768px以上→PC版、未満→モバイル版）

## 18. よくある質問と対処法（ガイドAI用）

### Q: スケジュールが生成できない（解なし）
**A**: 以下を順に確認:
1. 不可日が多すぎないか → 一部をソフト化
2. 勤務間隔が厳しすぎないか → 間隔を下げる
3. スコア下限が高すぎないか（特に外部枠使用時）→ 下限を下げる
4. 医師数が少なすぎないか → 医師を追加

### Q: 外部枠を使いたい
**A**: 設定 → 当直枠の範囲 → 「一部の日程」を選択。外部枠数（非常勤の日数）か勤務日数（自チームが担当する日数）のどちらかで指定。

### Q: 特定の日だけ外部にしたい
**A**: 設定 → カレンダーで外部にする日をタップ。日曜・祝日は日直/当直/終日を選べる。

### Q: D&Dで入れ替えられない
**A**: 制約違反がある場合は赤色で表示される。強制モードをONにすると制約を無視して操作可能。

### Q: 日当直モードで日直だけ入れ替えたい
**A**: 強制モードをONにすると、日直と当直を個別に操作可能。通常モードではペアで移動する。

### Q: ソフト化したのに反映されない
**A**: ソフト化後に「生成」を再実行する必要がある。D&D画面ではハード制約のままチェックされる（保存時に違反がわかる）。

### Q: 前月の当直と連続してしまう
**A**: 前月のシフトがDBに保存されているか確認。前月の当直表が未保存の場合、月マタギチェックが効かない。

### Q: メールアドレスを登録したい
**A**: ログイン後、アカウント設定（歯車アイコン）→「メールアドレス」セクションで登録・変更可能。メアドを登録するとメールアドレスでもログインできるようになる。

### Q: 病院名を変更したい
**A**: メールアドレスを先に登録してから、アカウント設定で変更可能（API: PUT /api/auth/name）。※現時点ではUI未実装、APIのみ。

### Q: メールアドレスでログインできない
**A**: 新規登録時にメアドを入力した方、またはアカウント設定でメアドを登録した方のみ対応。ログイン画面で病院名の代わりにメアドを入力すればOK。

## 19. AIガイド（チャットアシスタント）

### 概要
- `/app` と `/dashboard` の右下にフローティングチャットボタン
- クリックするとスライドインパネル（PC: 右側400px / モバイル: 全画面）が開く
- 当直表の作り方や設定について自然言語で質問できる

### 仕組み
- バックエンド `POST /api/guide/chat` がClaude APIを呼び出し
- システムプロンプトにこの仕様書 + レシピ集 + ユーザーの現在設定を注入
- 会話履歴はブラウザ内で管理（ページ離脱でリセット）
- 利用統計に `guide_chat` イベントとして記録

### 使い方
- 「月2回だけにしたい」「解なしになる」など、困りごとを自然に入力
- AIが既存の設定の組み合わせで解決策を提案
- 対応できない要望は代替案（ワークアラウンド）を提示
- 要望がある場合は「開発者に送信する」ボタンで直接フィードバック可能

### インサイト収集（内部機能）
- 全ての質問をカテゴリ分類（使い方/トラブル/改修要望/運用相談/その他）
- AI回答も含めて記録・集計
- /admin/dev でカテゴリ別集計・改修要望ランキングを確認可能
