## フェーズ1：ランディングページ（`/`）

### 目的
- 初見ユーザーが「何のアプリか」を理解できる
- ログイン・新規登録への導線を提供する

### 構成
```
ヘッダー
  └── [ログイン] [新規登録]

ヒーローセクション
  └── キャッチコピー + サービス概要 + [無料で試す] [ログイン]

こんな課題ありませんか？（ペインポイント）

解決策（仕組みの説明）

主な機能紹介
  - 自動スケジュール生成
  - ドラッグ&ドロップ編集
  - 医師ごとの不可日設定（マジックリンク）
  - 公平なスコア管理

デモ体験セクション → /demo へ誘導

登録CTA（ページ中間・最下部）
  └── [無料で試す] [デモを見る]
```

## フェーズ2：`/app`（初心者向けメイン画面）

### レイアウト

```
┌──────────────────────────────────────────────┐
│  ヘッダー  [設定]  [スケジュール生成]  [保存]   │
├──────────────────────────────────────────────┤
│                                              │
│  スケジュール表 + D&D                         │
│  （生成前: 「まず設定から始めましょう」案内）    │
│                                              │
└──────────────────────────────────────────────┘
```

### 設定UIの構造（2層）

```
[設定] クリック
→ フルスクリーンモーダル（設定メニュー一覧）
   ├── ルール設定（ハード制約）
   ├── 重み設定（ソフト制約）
   ├── 医師管理・個別不可日・固定不可曜日
   └── 目標スコア設定

各項目クリック
→ 右からドロワーが開く（編集UI）
```

### オンボーディングのトリガー
- 各ドロワーを**初めて開いたとき**に説明モーダルを表示
- スケジュール生成を**初めて実行したとき**に生成結果の見方を説明
- `localStorage` で「説明済みフラグ」を管理

### ログイン後の遷移
- デフォルト: ログイン → `/app`
- ユーザー設定で `/dashboard` を初期画面に変更可能（`system_settings` に保存）

## フェーズ3：セットアップウィザード

### 発動条件
初回ログイン時（`system_settings` に `setup_completed` キーが存在しない場合）

### 質問フロー

```
Step 1: 日当直の形式
  → 「日直と当直は別々ですか？同じ人が担当しますか？」
  → holiday_shift_mode を設定

Step 2: 医師人数の入力
  → 「何人で当直を回していますか？」
  → 入力した人数分の仮医師（「医師1」「医師2」...）を自動生成

Step 3: シフトスコア上限・下限の設定
  → 「1ヶ月に担当できる最大・最小の当直数（スコア）は？」
  → min_score / max_score を設定

Step 4: 基本ルールの設定
  → 「当直後は何日空けますか？」（interval_days）
  → 「土曜当直は月何回まで？」（max_saturday_nights）

Step 5: 完了
  → 「設定完了！まずデモ生成を試してみましょう」
  → /app にリダイレクト・スケジュール生成を促す
```

## フェーズ4：オンボーディング

### 管理方法
- `localStorage` のキー `onboarding_seen` にセクション名を配列で記録
- 例: `["rules", "weights", "doctors", "generate"]`

### 各セクションの説明内容

| セクション | 説明ポイント |
|-----------|------------|
| ルール設定 | 「守らなければならない制約です。まず当直間隔と上限から設定しましょう」 |
| 重み設定 | 「できれば守りたい優先度です。数値が大きいほど強く考慮されます」 |
| 医師管理 | 「医師を追加して、不可日や目標スコアを設定できます」 |
| スケジュール生成 | 「設定した制約に基づいて最適なシフトを自動で作ります」 |
| D&D編集 | 「生成後にドラッグ&ドロップで手動調整できます」 |

## フェーズ6：公開デモ（`/demo`）

### 仕様
- ログイン不要・認証なし
- DB への書き込み一切なし（最適化APIのみ呼び出し、結果はメモリのみ）
- 設定できる項目: 医師人数・基本ルール・当直形式
- 保存・医師名変更・不可日設定は**不可**
- 「本番で使うには登録を」のCTAを常に表示

### UI
- LPに埋め込み or `/demo` 独立ページ
- シンプルな入力フォーム → 生成 → 結果表示のみ
