# 最適化ロジック（OnCallOptimizer）

ファイル: `backend/services/optimizer.py`

## 概要
Google OR-Tools の **CP-SAT ソルバー**を使って当直スケジュールを自動生成する。
ハード制約（絶対条件）とソフト制約（重み付きペナルティ）の2層構造で最適解を探索する。

---

## スコアリング（シフト種別ごとの負荷スコア）

| シフト種別 | スコア |
|---------|------|
| 日直（日曜・祝日の午前） | 0.5 |
| 当直・夜間（平日） | 1.0 |
| 当直・夜間（土曜） | 1.5 |
| 当直・夜間（日曜・祝日） | 1.0 |

**スコア範囲目安:** 0.5 〜 4.5（月間）

---

## ハード制約（HardConstraints）

違反した場合は解なし（infeasible）になる絶対条件。

| 制約名 | 内容 |
|---------|------|
| 土曜夜間最大数 | 1名の医師が担当できる土曜夜間の月内上限 |
| 日曜・祝日最大勤務数 | 1名が担当できる日祝シフトの月内上限 |
| 連続勤務間隔 | 前のシフトから何日空けるか（デフォルト: 最低1日） |
| 不可日の尊重 | `is_fixed=True` の不可日はハード制約として強制 |
| ロック済みシフト | フロントエンドで固定したシフトは変更不可 |
| `holiday_shift_mode` | `"split"`（デフォルト）= 日直と当直を別々に割り当て / `"combined"` = 同一人物が日当直（スコア1.5）を担当 |

---

## 日当直モード（`holiday_shift_mode: "combined"`）

日曜・祝日に「日直」と「当直」を同一人物が担当するモード。

| モード | 日祝スロット | スコア |
|-------|-----------|------|
| `split`（デフォルト） | 日直（0.5pt）+ 当直（1.0pt）を別人 | 合計1.5pt（2名） |
| `combined` | 日当直（1.5pt）を1名が担当 | 1名に1.5pt |

- combined モードでは `day_shift == night_shift`（同一医師ID）でスケジュールを返す
- フロントエンドの「日当直」バッジで視覚的に識別
- DB保存時は `day` と `night` の2レコードとして同じ医師IDで保存（スキーマ変更不要）

---

## ソフト制約と重み（ObjectiveWeights）

ペナルティとして目的関数に加算される条件。値が大きいほど優先度が高い。

| パラメータ名 | デフォルト値 | 意味 |
|---------|------|------|
| `gap5` | 100 | `interval_days+1` 日ギャップをペナルティ（動的：interval_days=4なら5日） |
| `gap6` | 50 | `interval_days+2` 日ギャップをペナルティ（動的：interval_days=4なら6日） |
| `sat_month_fairness` | 100 | 同月内の土曜夜間回数を医師間で均等化 |
| `sat_consec` | 80 | 土曜夜間の連続月勤務 + 土曜夜間上限到達を抑制 |
| `score_balance` | 30 | 医師間のスコアバランスを均等化 |
| `target` | 10 | 目標スコアへの近似 |
| `sunhol_fairness` | 200 | 日曜・祝日シフトの公平な分配 |
| `sunhol_3rd` | 80 | 第3日曜・祝日の分散 |
| `month_fairness` | 100 | 月次スコア公平性 |
| `past_sat_gap` | 10 | 過去の土曜勤務間隔調整 |
| `past_sunhol_gap` | 5 | 過去の日曜祝日勤務間隔調整 |
| `soft_unavailable` | 100 | 不可日の回避優先度（`respect_unavailable_days=OFF` 時は全不可日がソフトペナルティになる） |

### 不可日の扱いについて

- `respect_unavailable_days=ON`（デフォルト）: 不可日・不可曜日をハード制約として強制。 `is_soft_penalty=True` のエントリのみソフトペナルティ扱い。
- `respect_unavailable_days=OFF`: 全不可日・不可曜日エントリが `soft_unavailable` 重みのソフトペナルティに切り替わる。重みが 0 の場合は完全に無視される。

### gap 重みの動的化

`gap5` / `gap6` ペナルティは `interval_days` に連動して動的に変化する：

| interval_days | gap5 が対象とする間隔 | gap6 が対象とする間隔 |
|------------|-------------------|-------------------|
| 4（デフォルト） | 5日 | 6日 |
| 3 | 4日 | 5日 |
| 5 | 6日 | 7日 |
| None（OFF） | 5日（フォールバック） | 6日（フォールバック） |

これにより「ハード制約の1つ上の間隔」が常にペナルティ対象となり、上限ぴったりの割り当てが好まれる問題を防ぐ。

## 主なメソッド

| メソッド | 役割 |
|---------|------|
| `build_model()` | CP-SATモデルを構築（制約・変数・目的関数を設定） |
| `solve()` | ソルバーを実行して最適解を返す |
| `is_holiday(date)` | 指定日が祝日かどうかを判定 |
| `is_sunday(date)` | 指定日が日曜日かどうかを判定 |

---

## 履歴スコア計算（`optimizer_history.py`）

| 関数名 | 役割 |
|---------|------|
| `build_past_total_scores()` | DBの過去シフト履歴から医師ごとの累積スコアを計算 |
| `score_historical_shift()` | 個別のシフトのスコアを計算 |

過去スコアは最適化時に参照され、長期的な公平性を保つために使用される。

---

## 最適化リクエスト（OptimizeRequest）

フロントエンドから送られる主なパラメータ：

```
- target_year_month: 対象年月
- doctors: 医師リスト（id, name, スコア設定, 不可日等）
- locked_shifts: 固定シフト一覧
- holidays: 祝日リスト
- hard_constraints: ハード制約設定
- objective_weights: ソフト制約の重み設定
```
