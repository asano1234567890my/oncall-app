# 外部枠（外部医師スロット）実装計画

> 設計確定・未着手（2026-03-26）

## コンセプト

「全枠を常勤医師で埋めなくてよい」を実現する。
Optimizerの制約を `AddExactlyOne`（必ず1人配置）→ `AddAtMostOne`（0人 or 1人）に切り替えることで、空き枠を許容する。

## ユースケース

| パターン | 例 | 動作 |
|---------|-----|------|
| A. 特定日が外部医師確定 | 「毎週水曜は非常勤」「4/5は外勤の先生」 | その日をロック済み扱い → 残りを常勤で生成 |
| B. 外部医師の日数だけ決まっている | 「月4回は非常勤が入る」 | 必要枠数を4つ減らして生成 → 空き枠が4つ残る |
| C. 常勤の上限が厳しくて全枠埋まらない | 「10人×最大2回=20回、枠は25」 | 埋められる分だけ生成 → 残り5枠は空きで確定可能 |

パターンAは現在でもダミー医師+ロックで可能だが、外部枠方式なら医師リストを汚さずにクリーンに実現できる。

### 生成→確定フロー

1. 管理者が「外部医師の枠: 4回/月」と設定
2. 生成ボタン押下
3. Optimizerが常勤医師で埋められる枠だけ埋める
4. スケジュール表に：
   - 常勤医師名が入ったセル
   - 「外部」と表示された空き枠（薄いグレー背景）
5. 管理者がD&Dまたはタップで外部枠に名前を入力 or そのまま確定
6. 確定保存 → 外部枠は空き or 手入力名で保存

## バックエンド改修

### Optimizer (`backend/services/optimizer.py`)

```python
# 現在（905行目）:
self.model.AddExactlyOne(self.night_shifts[(d, day)] for d in doctors)

# 変更後:
if external_slots_enabled:
    self.model.AddAtMostOne(self.night_shifts[(d, day)] for d in doctors)
else:
    self.model.AddExactlyOne(self.night_shifts[(d, day)] for d in doctors)
```

- `external_slot_count` パラメータで空き枠の数を制御
- 「最低でも (全枠 - external_slot_count) 枠は埋める」制約を追加
- `external_fixed_dates` で特定日を生成対象外にする（ロック済み扱い）
- 同様に907行目の `day_shifts` も同じ変更
- 診断ロジック（1530, 1532行目）も同様に対応

### リクエストスキーマ (`backend/schemas/optimize.py`)

```python
external_slot_count: int = 0          # 外部医師の枠数（0=全枠埋める）
external_fixed_dates: list[str] = []  # 外部医師が入る確定日（YYYY-MM-DD）
```

### 設定永続化

`optimizer_config` KV内に保存（既存の仕組みを流用）:
```json
{
  "external_slot_count": 4,
  "external_fixed_dates": ["2026-04-05", "2026-04-12"]
}
```
